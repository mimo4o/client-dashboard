### 第 1 部分：客戶資訊

詢問用戶：
1. **客戶名稱** — 品牌/公司名稱
2. **聯絡人姓名**
3. **聯絡人職稱** — 例如「品牌主理人」、「行銷經理」
4. **聯絡郵箱**
5. **所屬行業** — 例如「服飾業品牌」、「餐飲業」、「電商」

### 第 2 部分：項目資訊

詢問用戶：
1. **項目名稱** — 例如「品牌視覺重塑」、「電商網站開發」
2. **項目類型** — 例如「品牌設計」、「網站開發」、「社群行銷」
3. **一句話標語** — 用一句話概括項目核心價值
4. **項目摘要** — 2-3 句話描述要為客戶做什麼
5. **核心目標** — 列出 **剛好 4 個** 主要目標
6. **項目時長** — 例如「3 個月」、「6 個月」

---

### 第 3 部分：時間線階段

詢問用戶定義 **3-4 個階段**，每個階段包括：
1. **階段名稱**
2. **時間範圍** — 例如「第 1-4 週」
3. **具體任務** — 每個階段 3-4 個任務

如果用戶只給大綱，幫他們整理成清晰的階段。

## 收集完成後：生成文件

生成完整的 `src/lib/project-data.ts`，注意：

### 技術要求

- `status` 必須使用 `as const`（如 `"等待確認" as const`）
- `timeline[].status` 必須使用 `"upcoming" as const`
- `overview.objectives` 必須**剛好 4 個**
- `deliverables` 必須**剛好 4 個類別**
- `nextSteps` 必須**剛好 5 個步驟**
- 所有價格為整數（無小數）
- `subtotal` = 所有 `lineItems[].price` 的總和
- `total` = `subtotal` - `discount.amount`
- 圖片使用 Unsplash URL 或本地 `/filename.png` 路徑

### 同時更新 globals.css

找到 `src/app/globals.css` 的第 8-9 行：
```css
--color-accent: #7f161f;
--color-accent-dim: #7f161f10;
```
替換為用戶的品牌色。dim 版本是同一個色碼後面加 `10`（10% 透明度）。

### 完成後告訴用戶

```
npm install
npm run dev
```

然後打開 http://localhost:3000 查看儀表板。

部署：推送到 GitHub 並連接 Vercel，或執行 `npx vercel`。
